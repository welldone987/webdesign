# 项目规范摘要

完整、可执行的开发规范已合并到 `AGENTS.md`。本文只保留项目状态与常用流程摘要，便于快速 review。

## 项目定位

- 仓库类型：静态个人网站 monorepo。
- 当前重点：`apps/photography` 摄影集网站。
- 后续预留：`apps/home` 主页网站、`apps/resume` 简历网站。
- 技术栈：React + Vite + TypeScript + Tailwind CSS + Framer Motion。
- 部署：Cloudflare Pages；摄影站 Root directory 为 `apps/photography`，构建命令 `npm run build`，输出目录 `dist`。
- 不引入后端、数据库、登录系统或访问统计。

## 当前摄影站

- 页面流程：主页 -> 引导页 -> 展示页。
- 四个摄影主题：`暖`、`湛`、`盛`、`郁`。
- 展示页：左侧目录，右侧瀑布流。
- 图片详情：背景虚化，上层二级界面，左侧详细信息，右侧大图。
- 缺失 EXIF 字段显示 `:信息已消失`，并显示“不过回忆还在”。
- 视觉语言：米白偏白底色、黑色点缀、克制莫兰第重点色、少量 SVG 装饰、衬线体信息展示。

## 目录边界

```text
apps/
  photography/
  home/
  resume/
资源/  # 外部素材暂存区，不参与构建和版本管理
```

`资源/` 不得被代码、样式、测试或构建流程直接引用。站点只引用复制或生成到 `apps/photography/public/images/photography/` 的资产。

## 图片与数据

- 照片元数据：`apps/photography/src/data/photos.json`。
- 站点图片：`apps/photography/public/images/photography/`。
- 图片准备脚本：`apps/photography/scripts/prepare-photos.mjs`。
- 校验脚本：`apps/photography/scripts/validate-photos.mjs`。
- 单张站点图片不得超过 15MB。
- `prepare:photos` 从 `资源/摄影图片` 读取四个主题文件夹；不超过 15MB 的源图直接复制，超过 15MB 的源图低损失压缩到 15MB 内。

## 常用命令

```bash
npm install
npm run dev:photography
npm --workspace @personal-websites/photography run prepare:photos
npm run validate:photos
npm run typecheck:photography
npm run build:photography
```

最低验证：

```bash
npm run validate:photos
npm run typecheck:photography
npm run build:photography
```
